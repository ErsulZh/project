NEWSPULSE - ПОДРОБНОЕ ОПИСАНИЕ ДЛЯ ПЕРЕДАЧИ ПРОЕКТА
==================================================

1) ЧТО ЭТО ЗА ПРОЕКТ
--------------------
NewsPulse - это fullstack веб-приложение для демонстрации аналитики посещаемости
новостного сайта и прогнозирования трафика.

Проект показывает полный цикл:
- сбор данных (посещения и клики),
- хранение данных,
- вычисление статистики,
- прогнозирование,
- визуализация на графиках,
- экспорт аналитики в CSV.

Проще говоря: это готовый учебный/демо-аналитический продукт "под презентацию".


2) ДЛЯ ЧЕГО ПРОЕКТ НУЖЕН И ГДЕ ПРИГОДИТСЯ
-----------------------------------------
Подходит для:
- учебной защиты (курсовая, диплом, проектная работа);
- портфолио fullstack-разработчика;
- демонстрации навыков по аналитике данных и UX/UI;
- основы MVP для контентных сайтов/медиа;
- базы для будущего продакшн-решения.

Примеры применения:
- аналитика посещаемости новостного портала;
- аналитика трафика внутреннего корпоративного портала;
- учебный стенд для темы "прогнозирование временных рядов".


3) ИДЕЯ ПРОЕКТА В ОДНОМ АБЗАЦЕ
------------------------------
Пользователь открывает сайт, система фиксирует посещение, сохраняет данные по дням,
строит графики, считает недельные агрегаты, показывает тренды и прогноз на 7 дней
с доверительным коридором. Администратор видит визуальную аналитику, может вручную
добавлять данные и экспортировать результаты.


4) ЧТО РЕАЛИЗОВАНО ФУНКЦИОНАЛЬНО
--------------------------------
Главная:
- Лента новостей.
- Автоучет посещения при открытии страницы.
- Учет кликов по карточкам.
- Ручное добавление посещений на дату.

Админ-панель:
- Карточки метрик (сегодня, всего, число дней, среднее).
- График факта и прогноза по дням.
- Доверительный коридор (lower/upper), закрашенный на графике.
- Индикатор тренда (цветная стрелка + подпись).
- Недельный bar chart (среднее по неделям).
- Тепловая карта по дням недели (7xN).
- Блок "Прогноз по дням" с процентной дельтой.
- Кнопка "Скачать данные (CSV)".
- Порог оси Y сохраняется между перезагрузками.

Дополнительно:
- Отдельная страница "Как работает алгоритм".


5) АРХИТЕКТУРА (ПРОСТО И ПОНЯТНО)
---------------------------------
Состоит из 2 частей:

A) BACKEND (Python/Flask)
- Предоставляет API для записи и чтения данных.
- Считает агрегаты и прогноз.
- Хранит данные в файле backend/data.json.

B) FRONTEND (React/Vite)
- Показывает интерфейс и графики.
- Отправляет запросы в API.
- Отвечает за UX/UI и интерактивность.

Связь:
- Frontend делает запросы на /api/...;
- Vite proxy перенаправляет эти запросы на Flask.


6) ИСПОЛЬЗОВАННЫЕ ТЕХНОЛОГИИ И ЗАЧЕМ КАЖДАЯ
--------------------------------------------
BACKEND:
1. Python
   - Основной язык для серверной части.
   - Удобен для быстрой разработки аналитики и API.

2. Flask
   - Легкий веб-фреймворк.
   - Используется для эндпоинтов (visit, stats, forecast и т.д.).

3. Flask-Cors
   - Разрешает междоменные запросы между frontend и backend.
   - Нужен, чтобы не было проблем CORS в разработке.

FRONTEND:
1. React
   - Построение интерфейса из компонентов.
   - Удобно управлять состоянием и переиспользовать UI.

2. React DOM
   - Рендер React-компонентов в браузер.

3. React Router DOM
   - Маршрутизация между страницами:
     /, /admin_123, /about.

4. Vite
   - Быстрый dev-сервер и сборка.
   - Улучшает DX (скорость запуска/обновления).

5. @vitejs/plugin-react-swc
   - Быстрая трансформация React-кода (через SWC).

6. Chart.js
   - Библиотека графиков (line, bar).
   - Основа визуализации аналитики.

7. react-chartjs-2
   - React-обертка над Chart.js.
   - Упрощает работу с графиками в компонентах.

8. Framer Motion
   - Анимации появления/переходов.
   - Делает интерфейс "живым" и более презентабельным.

9. react-datepicker
   - Выбор даты в форме ручного ввода посещений.

10. date-fns
    - Работа с датами (локали и форматирование).

ИНСТРУМЕНТЫ:
1. concurrently
   - Одновременный запуск frontend и backend одной командой.


7) API (ЧТО ВОЗВРАЩАЕТСЯ И ДЛЯ ЧЕГО)
------------------------------------
POST /api/visit
- Учитывает посещение за текущий день.
- Использует cookie visitor_id, чтобы один посетитель не увеличивал счетчик
  бесконечно при обновлениях страницы.

POST /api/visit/manual
- Ручное добавление посещений на выбранную дату.
- Для демонстрации, тестовых сценариев и "дорисовки" данных.

POST /api/click
- Учет клика по новости.

GET /api/stats
- Базовая агрегированная статистика:
  dailyVisits, totalVisits, daysCount, averagePerDay, clicks.

GET /api/stats/weekly
- Недельная аналитика:
  avg за неделю + changePct к предыдущей.

GET /api/forecast
- Прогноз на 7 дней:
  date, value, lower, upper, trend.
- Также возвращает tsForecast как отдельный расчет.

GET /api/health
- Проверка, что сервер работает.


8) КАК РАБОТАЕТ ПРОГНОЗ (ПОНЯТНОЕ ОБЪЯСНЕНИЕ)
----------------------------------------------
Основной прогноз (forecast_week):
1. Берет последние данные посещений (до 30 точек).
2. Чистит аномалии (слишком выбивающиеся значения).
3. Считает тренд (линейная регрессия с весами последних дней).
4. Учитывает сезонность по дням недели.
5. Применяет экспоненциальное сглаживание.
6. Добавляет momentum (сравнение последних и предыдущих периодов).
7. Для каждого будущего дня формирует:
   - value (прогноз),
   - lower/upper (доверительный интервал),
   - trend (up/down/stable).

Зачем это нужно:
- тренд улавливает общее направление (рост/падение),
- сезонность учитывает поведение по дням недели,
- сглаживание уменьшает шум,
- интервалы дают представление о риске/неопределенности.


9) ВИЗУАЛИЗАЦИЯ И UX/UI
-----------------------
Почему интерфейс подходит для демонстрации:
- ключевые метрики "сверху" для быстрого понимания;
- line chart + confidence band выглядят как профессиональная аналитика;
- weekly chart дает обзор в более крупном масштабе;
- heatmap мгновенно показывает паттерны;
- tooltips дают детали по точке;
- анимации улучшают восприятие на защите.

11) КАК ЗАПУСТИТЬ
-----------------
Шаги:
1. Установить npm-зависимости:
   npm install
   cd frontend
   npm install
   cd ..

2. Установить Python-зависимости:
   pip install -r backend/requirements.txt

3. Запустить:
   npm start

После запуска:
- frontend: http://127.0.0.1:5173
- backend:  http://127.0.0.1:5000


12) ОГРАНИЧЕНИЯ ТЕКУЩЕЙ ВЕРСИИ
------------------------------
- Хранение в JSON, а не в СУБД.
- Нет авторизации и разграничения ролей.
- Нет набора автотестов и CI.
- Нет production-деплоя "из коробки".

Это нормально для учебного/MVP уровня.


13) КАК РАЗВИВАТЬ ПРОЕКТ ДАЛЬШЕ
-------------------------------
