# REPORT: NewsPulse (актуальная версия)

## 1) Кратко о проекте

**NewsPulse** — это fullstack-проект (frontend + backend) для демонстрации аналитики посещаемости новостного портала.

Проект решает сразу несколько задач:
- считает посещения и действия пользователя;
- показывает статистику в удобном UI;
- строит прогноз посещаемости на будущие дни;
- визуализирует доверительный коридор прогноза;
- дает данные для презентации (weekly-аналитика, heatmap, страница с описанием алгоритма, экспорт CSV).

---

## 2) Что реализовано в текущей версии

### Пользовательская часть
- Главная страница с карточками новостей.
- Автоматический учет посещения при открытии сайта.
- Клики по новостям сохраняются в аналитике.
- Форма ручного добавления посещений на выбранную дату (только не в будущее).

### Админ-панель аналитики
- Карточки ключевых метрик: сегодня, всего, число дней, среднее.
- График "Посещения по дням и прогноз" с:
  - фактическими данными;
  - прогнозной линией;
  - доверительным коридором (lower/upper).
- Индикатор тренда рядом с заголовком графика (стрелка вверх/вниз/нейтрально + подпись).
- Недельный bar chart (среднее по ISO-неделям + % к предыдущей неделе в tooltip).

- Секция "Прогноз по дням" с:
  - значением прогноза;
  - дельтой в процентах к предыдущему дню.
- Экспорт данных в CSV (история + прогноз + lower/upper).
- Порог оси Y сохраняется в `localStorage` и восстанавливается после перезагрузки.

## 4) API: детально

### `POST /api/visit`
Назначение:
- увеличить счетчик посещений за текущий день.

Особенность:
- используется cookie `visitor_id`;
- один и тот же посетитель учитывается максимум 1 раз в день.

### `POST /api/visit/manual`
Назначение:
- добавить указанное количество посещений на выбранную дату.

Валидация:
- дата обязательна;
- формат даты должен быть ISO;
- дата не может быть из будущего;
- count > 0.

### `POST /api/click`
Назначение:
- учесть клик по новости.

### `GET /api/stats`
Возвращает:
- `dailyVisits`,
- `totalVisits`,
- `daysCount`,
- `averagePerDay`,
- `clicks`.

### `GET /api/stats/weekly`
Возвращает недельные агрегаты:
- `isoYear`, `isoWeek`,
- `startDate`, `endDate`,
- `avg` (среднее за неделю),
- `changePct` (изменение к предыдущей неделе).

### `GET /api/forecast`
Возвращает:
- `prediction` — массив прогноза на 7 дней (каждый элемент содержит `value`, `lower`, `upper`, `trend`);
- `average` — среднее по горизонту прогноза;
- `tsForecast` — отдельный однодневный прогноз из `forecast_visits`.

## 5) Алгоритмы прогнозирования (backend/forecasting.py)

В проекте есть два алгоритма.

### 5.1 `forecast_week(...)` — основной недельный прогноз
Используется в `/api/forecast` для массива `prediction`.

Этапы:
1. Парсинг и подготовка:
   - сортировка по дате;
   - последние 30 записей;
   - отрицательные значения не допускаются.
2. Очистка аномалий:
   - правило `2 * std`.
3. Взвешенный линейный тренд:
   - веса `0.85^(...)`, последние дни важнее.
4. Сезонность:
   - коэффициенты по дням недели.
5. Экспоненциальное сглаживание:
   - `alpha = 0.6`.
6. Momentum:
   - сравнение последних и предыдущих окон.
7. Прогноз по дням:
   - формируется `value`, `lower`, `upper`, `trend`.
8. Ограничение значений:
   - защита от отрицательных и слишком больших значений.

### 5.2 `forecast_visits(...)` — отдельный TS-стиль прогноз на 1 день
Используется для поля `tsForecast` (дополнительная аналитика в API).

## API backend (Flask)

Файл `backend/app.py` реализует следующие эндпоинты (все начинаются с префикса `/api`):

- **`POST /api/visit`**
  - Назначение: увеличить счётчик посещений для **текущего дня**.
  - Логика:
    - читается `data.json`;
    - берётся текущая дата `YYYY-MM-DD`;
    - в секции `"visits"` увеличивается значение для этой даты;
    - обновлённые данные сохраняются обратно в файл.
  - Ответ:
    ```json
    {
      "ok": true,
      "date": "2026-05-05",
      "count": 5
    }
    ```

- **`POST /api/click?id=NEWS_ID`**
  - Назначение: увеличить счётчик кликов по новости с заданным `id`.
  - Параметры:
    - `id` — идентификатор новости (например, `"news-1"`).
  - Логика:
    - читается `data.json`;
    - в секции `"clicks"` увеличивается значение для этого `id`;
    - данные сохраняются.
  - Ответ:
    ```json
    {
      "ok": true,
      "id": "news-1",
      "count": 3
    }
    ```

- **`GET /api/stats`**
  - Назначение: вернуть общую статистику.
  - Логика:
    - читается `data.json`;
    - `"visits"` приводятся к массиву по дням: `[{ "date": "...", "count": N }]`;
    - считается:
      - `totalVisits` — общее количество посещений;
      - `daysCount` — количество дней, по которым есть данные;
      - `averagePerDay` — среднее посещений в день;
    - `"clicks"` возвращаются как объект `"id" -> count`.
  - Пример ответа:
    ```json
    {
      "dailyVisits": [
        { "date": "2026-05-05", "count": 12 },
        { "date": "2026-05-06", "count": 8 }
      ],
      "totalVisits": 20,
      "daysCount": 2,
      "averagePerDay": 10.0,
      "clicks": {
        "news-1": 5,
        "news-2": 3
      }
    }
    ```

- **`GET /api/forecast`**
  - Назначение: вернуть **простой прогноз** посещаемости.
  - Алгоритм:
    - читаются все значения из `visits`;
    - вычисляется среднее `average = totalVisits / daysCount`;
    - берётся последняя дата, по которой есть данные;
    - строится прогноз на **7 следующих дней**: каждый день имеет значение `average`.
  - Пример ответа:
    ```json
    {
      "average": 10.0,
      "prediction": [
        { "date": "2026-05-07", "value": 10.0 },
        { "date": "2026-05-08", "value": 10.0 }
      ]
    }
    ```

- **`GET /api/health`**
  - Простой health‑check, возвращает `{"status": "ok"}`.

## Структура и формат `data.json`

Файл `backend/data.json` содержит простую структуру:

```json
{
  "visits": {
    "2026-05-05": 12,
    "2026-05-06": 8
  },
  "clicks": {
    "news-1": 5,
    "news-2": 3
  }
}
```

- `visits[date]` — сколько раз за этот день была открыта главная страница;
- `clicks[id]` — сколько раз была нажата карточка с данным `id`.

Все операции backend проходят через функции:

- `load_data()` — читает JSON в память;
- `save_data(data)` — сохраняет данные с отступами и `ensure_ascii=False`;
- `ensure_data_file()` — создаёт начальный файл, если он отсутствует.

## Логика frontend (React)

### Главная страница — `HomePage.jsx`

- При первом рендере вызывается `useEffect`, внутри которого отправляется запрос:
  - `POST /api/visit` — добавляет одно посещение для текущего дня.
- Список новостей (`NEWS`) — статический массив с `id`, `title`, `category`, `excerpt`.
- При клике по карточке новости:
  - вызывается `handleNewsClick(id)`;
  - отправляется `POST /api/click?id=...`;
  - backend увеличивает счётчик кликов для соответствующей новости.
- Визуально:
  - используется компонент `NewsCard` с плавными анимациями появления (`framer-motion`);
  - сетка карточек (`.news-grid`) адаптивна и меняет количество колонок в зависимости от ширины экрана;
  - есть блок hero с промо‑описанием и бейджем «Live-аналитика».

### Админ‑панель — `AdminPage.jsx`

- При открытии `/admin_123`:
  - с помощью `Promise.all` запрашиваются:
    - `GET /api/stats`;
    - `GET /api/forecast`;
  - состояния:
    - `loading` — пока идут запросы;
    - `error` — если возникла ошибка сети или статус не `2xx`;
    - `stats`, `forecast` — сохранённые ответы.
- Отображение:
  - карточки с **общими метриками**:
    - всего посещений;
    - количество дней;
    - среднее посещений в день;
